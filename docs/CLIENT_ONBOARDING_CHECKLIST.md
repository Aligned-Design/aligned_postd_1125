## Phase 1: Pre-Onboarding (Before Signup)

### Sales/Success Handoff

- [ ] **Client company name** captured in CRM
- [ ] **Primary contact** name and email confirmed
- [ ] **Website URL** verified as accessible
- [ ] **Industry/business type** identified (e.g., coffee shop, accountant, real estate, salon)
- [ ] **Client goals** documented (awareness, engagement, leads, sales)
- [ ] **Current pain points** noted (time, consistency, brand alignment, multi-platform)
- [ ] **Expected posting frequency** discussed
- [ ] **Preferred platforms** identified (Instagram, Facebook, LinkedIn, TikTok, Google Business)
- [ ] **Handoff meeting** scheduled (if applicable)
- [ ] **Welcome email** sent with signup link

### Expectations Set

- [ ] Client understands onboarding takes ~10 minutes
- [ ] Client knows they'll review an auto-generated Brand Guide
- [ ] Client knows first content will be generated within 24-48 hours
- [ ] Client understands trial limits (if trial user: 7 days, 2 posts)

### Signup Flow

## âš ï¸ Error States & Recovery Paths

When things go wrong during onboarding, here's how to recover:

### Website Scrape Fails (Phase 2)

**What the client sees:**
- Error message: "We couldn't access your website. This can happen if the site is temporarily unavailable or blocks automated access."
- Two options presented: "Try Again" or "Set Up Manually"

**Recovery options:**
| Option | When to Use | Who Owns |
|--------|-------------|----------|
| **Retry scrape** | Transient network issue, site was temporarily down | Client (self-service) |
| **Manual setup** | Site blocks bots, complex SPA, requires login | Client (self-service) |
| **Success intervention** | Client stuck >2 hours, multiple retries failed | Success Team |

**Success team action:** If scrape fails twice, send personal outreach offering a 15-min setup call.

---

### Onboarding Abandoned Mid-Way (Any Phase)

**What the client sees:**
- On return: "Welcome back! Let's pick up where you left off." with a "Continue" button
- Dashboard shows onboarding incomplete banner

**Recovery options:**
| Trigger | What Happens | Who Owns |
|---------|--------------|----------|
| **Client returns within 24h** | Auto-resume from last step | System (automatic) |
| **Client doesn't return in 48h** | "Let's finish setting up" email sent | System (automated email) |
| **Client doesn't return in 5+ days** | Success team personal outreach | Success Team |

**Internal signal:** Dashboard shows "Onboarding Incomplete" status for this brand in Success team dashboard.

---

### Platform Connection Failure (Phase 4)

**What the client sees:**
- Error message: "We couldn't connect to [Platform]. This usually means the connection was cancelled or permissions weren't granted."
- "Try Again" button with link to platform-specific troubleshooting

**Recovery options:**
| Issue | Recovery | Who Owns |
|-------|----------|----------|
| **OAuth cancelled** | Client retries, ensures they click "Allow" | Client (self-service) |
| **Permissions denied** | Guide client to grant required permissions | Client + docs |
| **Token expired** | Client re-authenticates | Client (self-service) |
| **Platform outage** | Wait and retry later | Client (with status update) |

**Success team action:** If client attempts connection 3+ times without success, reach out with direct assistance.

---

### Content Generation Fails (Phase 5)

**What the client sees:**
- Error message: "Content generation is taking longer than expected. We're working on it."
- Option to "Retry" or "Skip for Now"

**Recovery options:**
| Issue | Recovery | Who Owns |
|-------|----------|----------|
| **AI service slow** | Retry after 30 seconds | System (automatic retry) |
| **AI service unavailable** | Fallback to deterministic default content | System (automatic fallback) |
| **Brand Guide incomplete** | Prompt client to complete Brand Guide first | System (UI guidance) |

**Note:** The system will never leave the client without content. Default content is generated if AI is completely unavailable.

---

### Recovery Ownership Summary

| Phase | Issue | First Response | Escalation |
|-------|-------|----------------|------------|
| Phase 2 | Scrape fails | System offers retry/manual | Success after 2 failures |
| Phase 2-8 | Abandoned mid-flow | Auto-resume + email | Success after 5 days |
| Phase 4 | Platform connection fails | Self-service retry | Success after 3 failures |
| Phase 5 | Content generation fails | Auto-retry + fallback | Support if persistent |
| Any | Technical error/crash | Error page with support link | Support immediate |
